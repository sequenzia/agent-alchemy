  ★ Insight ─────────────────────────────────────
  Wave-based topological execution: The 16 tasks form a dependency DAG (directed
   acyclic graph). By sorting into topological levels, we get 6 waves where all
  tasks within a wave can run in parallel since their dependencies are satisfied
   by earlier waves. This maximizes throughput — instead of running 16 tasks
  sequentially, we run them in 6 rounds.

  "Unblocks most" tiebreaker: Within each wave, tasks that appear in more
  blockedBy lists execute first. For example, task #4 (generate-tests skill)
  unblocks 3 downstream tasks (#7, #8, #14), so it runs before #6 (which only
  unblocks #7).
